# 🍽️ Telegram-бот викторина по меню

Интерактивный бот для изучения меню заведения в формате викторины.

## 🆕 Новая модульная архитектура

Проект разделён на независимые модули:

```
.
├── core/                    # 📦 Общее ядро (БД + бизнес-логика)
│   ├── database.py          # SQLite: пользователи, вопросы, статистика
│   └── quiz_engine.py       # Движок викторины (платформонезависимый)
├── bot_telegram/            # 🤖 Telegram-бот (только API)
│   ├── main.py              # Точка входа
│   ├── handlers.py          # Хендлеры сообщений
│   └── keyboards.py         # Клавиатуры
├── bot_vk/                  # 👥 VK-бот (заготовка)
│   └── main.py
├── webapp/                  # 🌐 Мини-приложение
│   └── index.html
├── menu_data.py             # Исходные данные (для миграции)
├── migrate_data.py          # Скрипт миграции в БД
├── run_bot.py               # Универсальный запуск
├── requirements.txt
└── .env.example
```

## ✨ Возможности

- **🌐 Мини-приложение (Web App)** — красивый интерфейс в Telegram
- **📊 База данных SQLite** — хранение вопросов, пользователей, прогресса
- **🔄 Исключение пройденных вопросов** — не повторяются уже отвеченные
- **📈 Статистика** — точность, серии, очки, прогресс по категориям
- **🗑 Сброс прогресса** — полный или по категории
- **🎲 Случайный вопрос** — быстрая викторина прямо в чате
- **👥 Мультиплатформенность** — ядро независимо от Telegram/VK

## 🚀 Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Настройка окружения

Скопируйте `.env.example` в `.env` и заполните:

```bash
cp .env.example .env
```

```env
BOT_TOKEN=123456789:ABCdefGHIjklMNOpqrsTUVwxyz
WEBAPP_URL=https://your-username.github.io/Bot-khinkalich/
```

### 3. Миграция данных (первый запуск)

```bash
python migrate_data.py
```

Или при запуске бота:
```bash
python run_bot.py --migrate
```

### 4. Запуск бота

```bash
# Telegram
python run_bot.py --platform telegram

# Или напрямую
python -m bot_telegram.main
```

## 📁 Структура базы данных

### Таблицы:

- **categories** — категории блюд (закуски, супы, хинкали...)
- **dishes** — блюда с описанием и ингредиентами
- **questions** — вопросы с вариантами ответов
- **users** — пользователи (универсальные, platform + external_id)
- **user_stats** — статистика (очки, точность, серии)
- **user_answers** — история ответов (для исключения повторов)

## 🎯 Как добавить новую платформу (VK, Discord и т.д.)

1. Создайте папку `bot_<platform>/`
2. Импортируйте функции из `core.quiz_engine`:

```python
from core import (
    register_user,      # Регистрация пользователя
    get_categories,     # Список категорий
    start_quiz_session, # Начать игру
    check_answer,       # Проверить ответ
    finish_quiz,        # Завершить игру
    get_user_full_stats,# Статистика
    reset_progress,     # Сброс прогресса
)
```

3. Реализуйте хендлеры сообщений платформы
4. Вся бизнес-логика уже готова!

## 📝 Добавление новых блюд

1. Откройте `menu_data.py`
2. Добавьте блюдо в список `DISHES`
3. Перезапустите миграцию:

```bash
python migrate_data.py
```
